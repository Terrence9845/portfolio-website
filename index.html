// Case Study 01
Outlook Email Not Sending
IT Documentation Outlook Email Config Authentication
+
Problem

User unable to send emails in Outlook — messages remained stuck in the Outbox. The user could still receive emails, but outgoing messages consistently failed to send.

Key Questions Asked
  • Are you connected to the internet?
  • Are you able to receive emails? (to isolate sending vs. full service failure)
  • Are there any error messages displayed?
  • When did the issue start? (to identify recent changes)
  • Have you recently changed your email password?
  • Is this affecting other users? (to determine if user-specific or system-wide)
Troubleshooting Steps
  • Confirmed device was connected to the internet
  • Verified Outlook was not in Offline Mode
  • Checked for stuck emails in the Outbox and attempted to resend or delete them
  • Closed and reopened Outlook to resolve any temporary application issues
  • Checked that email address, password, and server settings were correct
  • Removed and re-added the email account to refresh the configuration
  • Restarted the device to clear any remaining temporary system issues
Tools Used
Microsoft Outlook Network Settings Control Panel — Mail Settings
Root Cause & Resolution
✓ RESOLVED

Issue caused by an incorrect email password, preventing Outlook from authenticating with the server. The correct password was re-entered, allowing Outlook to reconnect and send successfully.

Reflection
This case highlights the importance of checking basic configuration issues — such as account credentials — before investigating more complex causes. Many email-related problems stem from simple authentication errors.
// Case Study 02
Slow Computer Performance
IT Documentation Windows Task Manager Malware Analysis
+
Problem

User reported their computer had become significantly slow during normal use — applications took an extremely long time to open, and the system would freeze or lag when performing basic tasks.

Key Questions Asked
  • When did the issue start? (to identify recent updates, installs, or changes)
  • Are specific applications slow, or the entire system?
  • Have you installed any new software recently?
  • How much storage space is available?
  • Have you noticed any unusual pop-ups or background activity? (to rule out malware)
  • When was the system last restarted?
Troubleshooting Steps
  • Opened Task Manager — reviewed CPU, memory, and disk usage, identified high-resource processes
  • Closed unnecessary background applications consuming system resources
  • Navigated to Startup tab in Task Manager and disabled non-essential startup programs
  • Checked disk storage and removed unnecessary files to free up space
  • Performed a full system scan using Windows Defender to rule out malware
  • Restarted the system to clear temporary processes and refresh performance
  • Checked for pending OS and driver updates to eliminate outdated software as a cause
Tools Used
Task Manager Startup Manager Disk Cleanup Windows Defender Windows Update
Root Cause & Resolution
✓ RESOLVED

Multiple background applications were consuming high CPU and memory usage, combined with several unnecessary startup programs. Unnecessary apps were closed, startup programs disabled, and disk space cleared.

Reflection
This case highlights the importance of monitoring system resources and managing startup applications. Many performance issues are caused by software and background processes rather than hardware failure. Malware scanning is a key step — malicious software can disguise itself as high resource usage.
// Case Study 03
Printer Not Printing
IT Documentation Print Spooler Device Manager Driver Management
+
Problem

A user reported their printer appeared connected but documents were not printing. Print jobs remained in the queue without completing. Some colleagues were unaffected, while others shared the same issue — suggesting a mixed device/network cause.

Key Questions Asked
  • Is the printer powered on? (confirm basic hardware state)
  • Are there any error messages on the printer or computer?
  • Are other users able to print successfully?
  • Is the printer connected via USB or network? (to identify physical vs. network issue)
  • When did the issue start? (to identify recent configuration changes or updates)
Troubleshooting Steps
  • Checked printer status — confirmed powered on, no paper jams, no empty ink
  • Checked print queue — cancelled all pending print jobs and restarted the printer
  • Restarted the printer to clear errors between the device and computer
  • Opened Windows Services, located the Print Spooler service, and restarted it
  • Confirmed the correct printer was set as the default printer
  • Updated and reinstalled printer drivers via Device Manager
Tools Used
Windows Printer Settings Print Queue Manager Windows Services Device Manager
Root Cause & Resolution
✓ RESOLVED

Multiple stuck print jobs had blocked the print queue. Clearing the queue, restarting the Print Spooler service, and reinstalling drivers restored full printing functionality.

// Case Study 04
User Cannot Connect to Wi-Fi
IT Documentation Networking ipconfig Network Settings
+
Problem

A user reported their laptop was unable to connect to the office Wi-Fi network. Other colleagues were connected without issues, suggesting the problem was device-specific rather than a network-wide fault.

Key Questions Asked
  • Are other devices able to connect? (to confirm device vs. network issue)
  • Are you receiving any specific error messages?
  • When did the issue start?
  • Have you recently changed your password? (outdated credentials are a common cause)
  • Are you working remotely or in the office? (signal strength and environment)
  • Is Wi-Fi enabled in your device settings?
Troubleshooting Steps
  • Confirmed Wi-Fi was enabled and Airplane Mode was disabled — a commonly overlooked issue
  • Restarted the laptop to resolve any temporary network or software glitches
  • Opened Network Settings, selected the Wi-Fi network, clicked Forget, then reconnected with the correct password
  • Used Command Prompt (ipconfig) to check IP configuration and confirm address assignment
  • Checked Device Manager for any driver issues with the wireless adapter
Tools Used
Windows Network Settings Command Prompt (ipconfig) Device Manager
Root Cause & Resolution
✓ RESOLVED

The device was storing an outdated Wi-Fi password after network credentials had been changed. Forgetting the network and reconnecting with the updated password resolved the issue immediately.

Reflection
This case highlights the importance of gathering information before troubleshooting. Many connectivity issues are caused by simple configuration problems rather than hardware failures — and asking the right questions upfront saves significant diagnostic time.
